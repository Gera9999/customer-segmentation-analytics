# Customer Segmentation Analytics (RFM + Clustering)

A professional customer analytics portfolio project using the **Online Retail II** (UCI / Kaggle) transactional dataset.

This repository demonstrates an end-to-end workflow used in marketing analytics:

- Transaction data cleaning and preparation
- Customer-level aggregation and revenue metrics
- **RFM analysis** (Recency, Frequency, Monetary) and rule-based segmentation
- **K-Means clustering** on standardized RFM features
- Automated business insights
- An **interactive Plotly dashboard** exported to HTML

---

## Dataset

**Online Retail II (UCI)** contains e-commerce transactions with these fields:

- `InvoiceNo`
- `StockCode`
- `Description`
- `Quantity`
- `InvoiceDate`
- `UnitPrice`
- `CustomerID`
- `Country`

Each row represents a product purchased in a transaction.

**Expected local path:** `data/online_retail.csv`

Note: The dataset file is intentionally **not committed** to the repository (size/licensing). Download it from Kaggle/UCI and place it at the path above.

## Methodology

### 1) Data Cleaning
Implemented in `src/data_cleaning.py`:

- Remove rows with missing `CustomerID`
- Remove negative/zero `Quantity` (returns / invalid rows)
- Remove rows with `UnitPrice <= 0`
- Convert `InvoiceDate` to datetime
- Create `Revenue = Quantity * UnitPrice`

### 2) Customer Aggregation
Implemented in `src/rfm_analysis.py` (`aggregate_customers`):

- Total Revenue
- Total Transactions (`InvoiceNo` unique count)
- Total Products Purchased (sum of `Quantity`)
- First Purchase Date
- Last Purchase Date

### 3) RFM Analysis
Implemented in `src/rfm_analysis.py`:

- **Recency**: days since last purchase (relative to dataset snapshot date)
- **Frequency**: number of transactions
- **Monetary**: total revenue

RFM scores are created using quantile-based binning (1–5). Recency is reversed (lower recency → higher score).

#### RFM Segments
Rule-based segments include:

- Champions
- Loyal Customers
- Potential Loyalists
- New Customers
- At Risk
- Hibernating

### 4) Clustering (K-Means)
Implemented in `src/clustering.py`:

- Standardize `Recency`, `Frequency`, `Monetary`
- Compute inertia for a range of $k$ (Elbow Method)
- Choose $k$ using a simple curvature heuristic
- Fit K-Means and assign a `Cluster` label per customer

